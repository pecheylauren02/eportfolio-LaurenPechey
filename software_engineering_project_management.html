Software Engineering Project Management

This module introduced the multidisciplinary role of the Software Engineering Project Manager, combining traditional project management practices with software engineering principles. It explored key areas including planning, estimation, resource management, requirements engineering, agile management, and collaborative design within software projects.

UNIT 1

Introduction to Software Engineering Project Management

During Week 1, I was introduced to the multidisciplinary role of the Software Engineering Project Manager. This unit explored the combination of traditional project management practices, including planning, estimation, and resource management, alongside core software engineering principles. It also examined technical responsibilities such as requirements engineering, agile project management, and collaborative design, establishing a foundational understanding of how these disciplines integrate within modern software development projects.

LEARNING OUTCOMES

  • Explain the differences between traditional project management and Software Engineering Project Management (SEPM).
  • Justify the need for SEPM in the planning and delivery of software projects.
  • Discuss the historical development of project management and software engineering.
  • Explain the relationship between Project Management, TOGAF, and the Software Development Life Cycle (SDLC).

REFLECTION

Reflecting on my learning in Unit 1: Introduction to Software Engineering Project Management, I apply the Rolfe et al. (2021) reflective model to critically examine my understanding of Software Engineering Project Management (SEPM) and its relevance to contemporary software projects.

What?

This unit introduced the multidisciplinary role of the Software Engineering Project Manager, distinguishing traditional project management from SEPM. Key topics included the historical evolution of both disciplines, commonly used project management tools and certifications, and the relationship between Project Management, TOGAF, and the Software Development Life Cycle (SDLC). Formative activities focused on team formation, assessment briefing, and establishing a group contract to support collaborative project delivery.

So What?

Critically, the unit demonstrated that traditional project management alone is insufficient for software projects characterised by uncertainty and evolving requirements. SEPM integrates technical practices such as requirements engineering and agile management with governance structures, reducing failure risks identified in large-scale projects (Sommerville, 2020; PMI, 2021; Kerzner, 2022).

Now What?

I will apply SEPM principles by critically evaluating SDLC choices, strengthening requirements management, and supporting transparent communication within my project team.

References:

Kerzner, H. (2022) Project management: a systems approach to planning, scheduling, and controlling. 13th edn. Hoboken: Wiley.

PMI (2021) A guide to the project management body of knowledge (PMBOK® Guide). 7th edn. Newtown Square, PA: Project Management Institute.

Rolfe, G., Freshwater, D. and Jasper, M. (2021) Critical reflection in practice: generating knowledge for care. 3rd edn. London: Palgrave Macmillan.

Sommerville, I. (2020) Software engineering. 10th edn. Harlow: Pearson.

UNIT 5

User Experience

During Week 5, I examined the concept of User Experience (UX) within the context of software development projects. The unit explored how different stakeholders influence and prioritise user experience, as well as the critical role of the Project Manager in facilitating and managing UX throughout the project lifecycle.

LEARNING OUTCOMES

  • Demonstrated an understanding of how user experience can be planned and prioritised within a software project.
  • Defined strategies for measuring and managing user experience.
  • Recognised how Test-driven Development approaches in Python can support a high-quality user experience.

REFLECTION

Reflecting on my learning in Unit 5: User Experience, I applied the Rolfe et al. (2021) reflective model to examine the role of user experience within software engineering projects and the responsibilities of the Project Manager in supporting effective UX.

What?

This unit explored usability principles, stakeholder involvement in user experience, and methods for monitoring and measuring UX across the project lifecycle. It also examined how Test-driven Development in Python can support improved user experience through higher software quality.

So What?

The unit demonstrated that user experience is a shared responsibility requiring structured planning and continuous evaluation. Research indicates that integrating UX into project management improves usability and user satisfaction (Sommerville, 2020; ISO 9241-210, 2019). The use of Test-driven Development and code quality tools further supports UX by reducing defects and enhancing reliability (Beck, 2020).

Now What?

I will apply these insights by embedding UX considerations into project planning and using test-driven practices to support continuous improvement.

References:

Beck, K. (2020) TDD by example. 2nd edn. Boston: Addison-Wesley.
ISO (2019) ISO 9241-210. Geneva: ISO.
Rolfe, G. et al. (2021) Critical reflection in practice. London: Palgrave.
Sommerville, I. (2020) Software engineering. Harlow: Pearson.
