Object Oriented Programming

This Module explores the principles and practices of Object Oriented Programming (OOP), focusing on designing and developing modular, reusable, and maintainable software.

UNIT 1

LEARNING OUTCOMES

  • Explore the waterfall and agile approaches to software development, with a focus on the implications of developing secure software using each.
  • Become acquainted with the Unified Modelling Language and how it can be used to support software development.
  • Gain a familiarity with the standards which are used by industry to develop secure software.
  • Appreciate the importance of developing a risk-aware culture within an organisation.

REFLECTION

REFLECTION

Reflecting on my learning in Unit 1: An Introduction to Python Programming and the Object-Oriented Programming Paradigm, I will employ the Rolfe et al. (2001) reflective model to critically evaluate my understanding and application of Object Oriented Programming (OOP) principles in Python.

What?

This unit provided a comprehensive introduction to the foundational principles of OOP, including abstraction, encapsulation, inheritance, and polymorphism, and their implementation in Python. I engaged in defining classes, creating objects, and managing data types and access modifiers, consolidating both theoretical knowledge and practical programming skills. Setting up the PyCharm IDE enabled structured experimentation with Python syntax, fostering a deeper understanding of how object-oriented structures facilitate modularity and code organisation.

So What?

This experience highlighted the significance of OOP in producing maintainable and scalable software. Hands-on exercises demonstrated how encapsulation and abstraction reduce complexity, while inheritance and polymorphism enhance code flexibility and reuse. Collaborative discussions and guided exercises reinforced my ability to translate theoretical concepts into functional implementations, emphasising disciplined coding practices and adherence to programming standards.

Now What?

Moving forward, I aim to apply these OOP principles to more sophisticated programming tasks, using UML to model class hierarchies and interactions effectively. I will continue to prioritise code readability, maintainability, and adherence to Python best practices, ensuring that future projects integrate both theoretical understanding and practical execution. This unit has provided a solid academic and technical foundation for advanced object-oriented development in subsequent modules and professional contexts.

Welcome to Week 2. In this unit, we will begin developing Python programs by first analysing the problem domain using UML diagrams, focusing on use case and state machine diagrams to inform program design.

  • Analyse software requirements through the creation of UML use case diagrams to model actor interactions.
  • Develop state machine diagrams to examine system behaviours and transitions.
  • Apply object-oriented principles in Python by managing data retention and variable assignment across class instances.

REFLECTION

Reflecting on my experiences in Unit 2, I will employ the Rolfe et al. (2001) reflective model to structure my insights on object-oriented analysis and its application to Python programming.

What?

This unit introduced the practical use of UML diagrams to analyse software systems before coding. I learned to construct use case diagrams to capture functional requirements and state machine diagrams to model dynamic behaviours within a system. These activities reinforced the relationship between analysis and program design, and highlighted how object-oriented principles guide variable management and data flow within Python classes.

So What?

Engaging with UML modeling and practical Python exercises enabled me to translate theoretical concepts into structured software designs. Understanding actor interactions and system behaviours deepened my comprehension of class design, object interactions, and data encapsulation. Collaborative discussions and practical tasks emphasised the importance of accurate analysis in guiding robust and maintainable Python programs, reinforcing the value of combining planning with implementation.

Now What?

Moving forward, I plan to integrate UML-based analysis into my programming workflow consistently, using use case and state machine diagrams to guide class design and object interactions. I will apply these insights to develop Python programs that are well-structured, maintainable, and aligned with object-oriented principles. Welcome to Week 3. In this unit, we will focus on the Unified Modelling Language (UML) as a standardized toolset to represent software systems prior to their implementation in Python.

LEARNING OUTCOMES

  • Understand the syntax and structure of common UML diagrams.
  • Analyse and apply UML models to support object-oriented analysis, design, and development.
  • Create UML diagrams to plan software systems effectively before coding in Python.

REFLECTION

Reflecting on my learning in Unit 3, I will employ the Rolfe et al. (2001) reflective model to evaluate my understanding and application of UML in object-oriented design.

What?

This unit introduced UML as a standardized method for visually representing software systems, allowing for structured analysis and design prior to coding. I explored various UML diagram types, including class, use case, and state machine diagrams, and practiced creating these using open-source tools. These activities reinforced my understanding of object-oriented concepts by illustrating how classes, objects, and interactions can be modelled before implementation in Python.

So What?

Engaging with UML diagrams improved my ability to plan and communicate software designs effectively. Practical exercises demonstrated how visual models facilitate understanding of complex systems and guide object-oriented development. Collaborating with peers and reviewing existing UML examples helped me recognise best practices and potential pitfalls, enhancing my analytical and design skills. This experience highlighted the importance of bridging theoretical analysis with practical programming tasks to develop maintainable and well-structured software.

Now What?

Moving forward, I plan to integrate UML modelling consistently into my programming workflow to support clear system design and implementation in Python. I will apply these diagrams to plan class hierarchies, object interactions, and state transitions, ensuring robust and maintainable code. This unit has strengthened my ability to connect analysis, design, and coding, providing a foundation for more advanced object-oriented programming tasks in future modules.

UNIT 5

Welcome to Week 5. This week we extend our exploration of Python classes, focusing on constructors, abstract classes, and the principles of modularity, inheritance, aggregation, and composition.

  • Apply inheritance, polymorphism, aggregation, and composition to enhance code modularity and reusability.
  • Understand the use of packages and namespaces to organise variables and classes effectively.
  • Implement Python programs that demonstrate advanced object-oriented programming techniques.

REFLECTION

Reflecting on Unit 5, I deepened my understanding of advanced object-oriented programming techniques and their application in Python.

What?

This unit taught me how to use constructors and abstract classes to structure Python programs effectively. I explored inheritance, polymorphism, aggregation, and composition, and examined how packages and namespaces support modularity and code organisation in larger systems.

So What?

Applying these concepts in practical exercises helped me see how advanced OOP principles improve maintainability, reusability, and scalability. Understanding the interplay between inheritance and modular design reinforced the importance of structuring programs to manage complexity and support collaboration.

Now What?

Moving forward, I will consistently apply constructors, abstract classes, and inheritance patterns in my Python programs. I will focus on designing modular, maintainable, and reusable code while leveraging namespaces and packages to manage complexity in larger projects. This unit has strengthened my capability to implement robust object-oriented designs in both academic and professional contexts.
