Secure Software Development

This Module covers best practices for building secure applications, focusing on threat modeling, secure coding techniques, and vulnerability assessments.

UNIT 1

LEARNING OUTCOMES

  • Explore the waterfall and agile approaches to software development, with a focus on the implications of developing secure software using each.
  • Become acquainted with the Unified Modelling Language and how it can be used to support software development.
  • Gain a familiarity with the standards which are used by industry to develop secure software.
  • Appreciate the importance of developing a risk-aware culture within an organisation.

REFLECTION

Reflecting on my experience in the Secure Software Development module, I will utilize the Rolfe et al. (2001) reflective model to structure my thoughts and insights.

What?

The module introduced me to crucial concepts in secure software development, particularly contrasting traditional methodologies like the waterfall model with modern agile approaches. I learned about the importance of flexibility in responding to changing requirements, especially regarding security considerations. The use of the Unified Modelling Language (UML) in our projects helped me visualize and decompose tasks effectively, creating flowcharts that mapped out software processes. This practical application was instrumental in understanding the design phase of software development. Additionally, the module emphasized fostering a risk-aware culture, highlighting that security is not just a technical issue but also a human one.

So What?

Participating in a development team allowed me to experience various roles, from coding to security testing. Allocating work effectively among team members was essential for maintaining progress and meeting deadlines. I found that taking on different responsibilities, such as developing secure APIs and conducting code reviews, enhanced my understanding of secure coding practices and how security can be integrated into each stage of the development lifecycle. Team meetings were vital for collaboration; they provided a platform to discuss challenges and share insights, reinforcing the need for open communication and collective problem-solving.

Now What?

Moving forward, I plan to implement the lessons learned in future projects. Emphasizing security from the outset, I will advocate for agile methodologies that allow for ongoing adaptation to security threats. I aim to integrate risk management into my workflow, ensuring that my future teams maintain a culture of security awareness. Moreover, I intend to further develop my skills in UML to improve my ability to model software deployments effectively. This module has solidified my understanding that secure software development is a continuous process that thrives on collaboration, adaptability, and a proactive security mindset.
